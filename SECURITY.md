# Security Policy

## Reporting a Vulnerability

If you discover a security issue in any of the skill files (e.g., a prompt pattern that could lead to code injection, an insecure code example, or a misconfigured security checklist), please report it by opening an [issue](https://github.com/deserteaglemjAEC/web-design-workflow/issues).

## Scope

This project contains markdown instruction files for Claude, not executable code. Security concerns are most likely to appear in:

- **Code examples** within skill files (e.g., `security-review.md`, `design-master.md`) that users may copy into their projects
- **Recommended patterns** that could be insecure if followed as-is
- **Dependencies** referenced in skill instructions (npm packages, Python packages)

## Response

We will acknowledge reports within 7 days and publish a fix as soon as practical.
